## Лабораторная работа №5  
**Тема:** Работа с формами. Аутентификация и регистрация пользователей в Django

### 1. Цель работы
- **Освоить** работу с Django‑формами (Form, UserCreationForm).
- **Реализовать** аутентификацию (login/logout) и регистрацию пользователей.
- **Добавить** в приложение опросов форму создания новых опросов с вариантами ответов.

### 2. Постановка задачи
- На основе примера блога/опросов из лекций реализовать приложение `polls`:
  - просмотр списка опросов и результатов голосования;
  - возможность проголосовать за один из вариантов;
  - создание нового опроса через HTML‑форму.
- Реализовать страницы:
  - **регистрации** пользователя;
  - **входа** (login);
  - **выхода** (logout) с разлогином по GET и редиректом на страницу входа.

### 3. Структура проекта
- **Проект**: `lab5proj`
- **Приложение**: `polls`
  - модели:
    - `Question(question_text, pub_date, author)` — вопрос опроса;
    - `Choice(question, choice_text, votes)` — вариант ответа с количеством голосов.
  - формы:
    - `PollCreateForm` — создание нового опроса (поле для заголовка и `textarea` с вариантами по одному на строку);
    - `RegisterForm` — регистрация пользователя на основе `UserCreationForm` (добавлено поле email).
  - основные представления (`views`):
    - `index` — список опросов;
    - `detail` — детализация опроса с вариантами ответа;
    - `results` — страница результатов голосования;
    - `vote` — обработка голоса (только для авторизованных);
    - `poll_create` — создание нового опроса (только для авторизованных);
    - `register_view` — регистрация пользователя;
    - `logout_view` — выход пользователя и редирект на страницу входа.
  - базовый шаблон `base.html` с меню навигации (опросы, новый опрос, вход/выход, регистрация).

### 4. Тестовый пользователь
- **Имя пользователя**: `student`  
- **Пароль**: `student12345`  
- Пользователь имеет права суперпользователя (доступ в `/admin/`).


### 5. Основные URL‑адреса
- Список опросов: `/polls/`
- Создание опроса: `/polls/create/` (требуется вход)
- Регистрация: `/register/`
- Вход: `/login/`
- Выход: `/logout/`
- Админ‑панель: `/admin/`

### 6. Краткие выводы
- Реализовано приложение с формой создания опроса, где варианты ответа задаются списком строк в `textarea`.
- Подключена встроенная система аутентификации Django (регистрация, вход, выход, защита страниц декоратором `login_required`).
- Интерфейс оформлен с использованием Bootstrap и общего базового шаблона, обеспечивающего удобную навигацию по страницам лабораторной работы.
